# Day 75: 测试执行与数据收集(2) - 性能压测与安全扫描

## 🎯 1. 核心风险与测试目标 (30分钟)

> **测试工程师视角**：性能和安全是系统质量的重要维度。如果测试不充分，会导致：
> - 系统在高负载下崩溃，影响用户体验和业务连续性
> - 安全漏洞被利用，造成数据泄露和合规风险
> - 性能瓶颈未被发现，上线后扩容成本高昂
> - 安全扫描覆盖不足，遗漏关键风险点

### 1.1 性能与安全风险

| 风险类型 | 风险场景 | 业务影响 | 测试应对 |
|---------|---------|---------|---------|
| 🔴 性能 | 高并发下响应超时 | 用户流失，交易失败 | 压力测试、容量规划 |
| 🔴 性能 | 内存泄漏导致OOM | 服务崩溃，数据丢失 | 稳定性测试、内存监控 |
| 🔴 安全 | SQL注入攻击 | 数据泄露，系统被控 | 注入测试、参数化查询验证 |
| 🔴 安全 | 越权访问 | 敏感数据暴露 | 权限测试、访问控制验证 |
| 🟡 性能 | 数据库连接池耗尽 | 服务不可用 | 连接池监控、超时配置 |
| 🟡 安全 | 敏感信息明文传输 | 中间人攻击 | 传输加密验证、证书检查 |

### 1.2 测试目标

**性能测试目标**：
1. **基准建立**：建立系统性能基线，支持后续回归对比
2. **瓶颈识别**：发现系统性能瓶颈，指导优化方向
3. **容量规划**：确定系统最大承载能力，支持扩容决策
4. **稳定性验证**：验证系统长时间运行的稳定性

**安全测试目标**：
1. **漏洞发现**：识别常见安全漏洞（OWASP Top 10）
2. **合规验证**：验证安全合规要求（等保、GDPR等）
3. **防护验证**：验证安全防护措施有效性
4. **风险量化**：评估安全风险等级，指导修复优先级

## 📚 2. 理论基础

### 2.1 性能测试类型

```
┌─────────────────────────────────────────────────────────────────┐
│                      性能测试类型金字塔                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│              ┌─────────────┐                                    │
│              │  压力测试    │  ← 超出正常负载，测试极限          │
│              │  (Stress)   │     目标: 发现崩溃点               │
│              └──────┬──────┘                                    │
│                     │                                           │
│           ┌─────────┴─────────┐                                 │
│           │     负载测试       │  ← 模拟正常/峰值负载             │
│           │    (Load Test)    │     目标: 验证性能指标           │
│           └─────────┬─────────┘                                 │
│                     │                                           │
│        ┌────────────┴────────────┐                              │
│        │        基准测试          │  ← 标准负载下的性能基线        │
│        │     (Benchmark)         │     目标: 建立性能基准         │
│        └─────────────────────────┘                              │
│                                                                 │
│   底层支持: 并发测试、容量测试、稳定性测试、配置测试              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 性能指标与SLA

| 指标类别 | 具体指标 | 典型SLA | 测量方法 |
|---------|---------|---------|---------|
| **响应时间** | 平均响应时间 | < 200ms | APM工具、日志分析 |
| | P95响应时间 | < 500ms | 百分位统计 |
| | P99响应时间 | < 1000ms | 百分位统计 |
| **吞吐量** | QPS/TPS | > 1000 | 压测工具统计 |
| **资源使用** | CPU使用率 | < 70% | 监控工具 |
| | 内存使用率 | < 80% | 监控工具 |
| | 磁盘I/O | < 80% | 监控工具 |
| **可用性** | 成功率 | > 99.9% | 健康检查 |
| | 错误率 | < 0.1% | 日志统计 |

### 2.3 OWASP LLM Top 10 安全测试

```
┌─────────────────────────────────────────────────────────────────┐
│                   OWASP LLM Top 10 (2025)                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  LLM01 │ Prompt注入              │ 恶意输入覆盖系统指令         │
│  LLM02 │ 不安全输出处理          │ 未过滤的输出导致XSS/注入      │
│  LLM03 │ 训练数据投毒            │ 恶意数据影响模型行为          │
│  LLM04 │ 模型拒绝服务            │ 资源耗尽攻击                  │
│  LLM05 │ 供应链漏洞              │ 依赖组件的安全风险            │
│  LLM06 │ 敏感信息泄露            │ 训练数据或Prompt信息泄露      │
│  LLM07 │ 不安全的插件设计        │ 插件调用安全风险              │
│  LLM08 │ 过度代理权限            │ Agent权限过大导致风险         │
│  LLM09 │ 过度依赖模型输出        │ 未验证的模型输出被直接使用    │
│  LLM10 │ 模型窃取                │ 模型被逆向或提取              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.4 安全测试方法

| 测试方法 | 描述 | 工具示例 | 适用场景 |
|---------|------|---------|---------|
| **静态分析** | 不运行代码，分析源代码 | SonarQube, Bandit | 代码审计 |
| **动态扫描** | 运行时测试，模拟攻击 | OWASP ZAP, Burp Suite | Web应用 |
| **依赖扫描** | 检查第三方依赖漏洞 | Snyk, Dependabot | 供应链安全 |
| **渗透测试** | 人工模拟真实攻击 | Metasploit, Cobalt Strike | 深度安全评估 |
| **模糊测试** | 随机输入发现异常 | AFL, Peach | 输入验证 |
