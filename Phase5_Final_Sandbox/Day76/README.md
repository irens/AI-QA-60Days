# Day 76: 性能压测深度实践(1) - 负载生成策略

## 🎯 学习目标

掌握性能测试中的负载生成策略，理解不同负载模型的适用场景，能够设计符合业务特征的负载模式。

## 📚 理论基础

### 1. 负载模型类型

#### 1.1 恒定负载模型（Constant Load）

```
并发用户数
    │
100 ├────────────────────────
    │
  0 └────────────────────────► 时间
         测试持续时间
```

**适用场景**：
- 系统基准容量验证
- 稳定性测试（长时间恒定负载）
- 简单的容量规划

**优缺点**：
- ✅ 简单易实现，结果易于分析
- ❌ 不符合真实业务流量特征

#### 1.2 阶梯负载模型（Step Load）

```
并发用户数
    │
300 ├────────────────────────
    │
200 ├────────┬───────────────
    │        │
100 ├────────┼────────┬──────
    │        │        │
  0 └────────┴────────┴──────► 时间
       阶段1   阶段2   阶段3
```

**适用场景**：
- 寻找系统性能拐点
- 确定最大承载容量
- 容量规划与扩展性评估

**优缺点**：
- ✅ 能够识别性能瓶颈点
- ❌ 需要较长时间完成测试

#### 1.3 脉冲负载模型（Spike Load）

```
并发用户数
    │
500 ├────────┐    ┌────────
    │        │    │
100 ├────────┼────┼────────
    │        │    │
  0 └────────┴────┴────────► 时间
         突发流量    突发流量
```

**适用场景**：
- 秒杀/抢购场景验证
- 突发流量应对能力测试
- 弹性伸缩触发验证

**优缺点**：
- ✅ 验证系统应对突发流量的能力
- ❌ 难以精确控制，结果波动较大

#### 1.4 波浪负载模型（Wave Load）

```
并发用户数
    │
200 ├──────┐      ┌──────┐
    │      │      │      │
100 ├──────┼──────┼──────┼───
    │      │      │      │
  0 └──────┴──────┴──────┴───► 时间
       高峰  低谷  高峰  低谷
```

**适用场景**：
- 模拟日常业务高峰低谷
- 电商大促周期模拟
- 验证资源回收与再利用

### 2. 负载生成关键参数

| 参数 | 说明 | 示例 |
|-----|------|------|
| 并发用户数(VU) | 同时模拟的用户数量 | 100, 500, 1000 |
| 思考时间(Think Time) | 用户操作间隔 | 1-5秒随机 |
|  Ramp-up时间 | 用户启动间隔 | 每秒启动10个用户 |
| 持续时间 | 负载持续时长 | 10分钟/30分钟/1小时 |
| 目标吞吐量 | 期望的TPS/QPS | 1000 TPS |

## � 关键要点

1. **负载模型选择原则**：根据业务特征选择，而非技术便利性
2. **真实业务数据**：基于生产流量分析设计负载参数
3. **渐进式加压**：避免瞬间高压导致系统直接崩溃
4. **监控联动**：负载生成与系统监控同步，实时观察响应
