# 07 · LLM-as-a-Judge：用 AI 评 AI，谁守门？

> 前面 01（幻觉检测）用了一个 LLM 裁判，09（用例验收）特意用了"可信参照"而非 AI。
> 这一篇收口那个反复出现的伏笔：**当你不得不用 LLM 当裁判时——裁判自己可不可信？**

群聊那句"测试也 AI 了，谁来守门"，最尖锐的版本就是:**评估也交给 AI 了，那谁来评估这个评估者？**

---

## 🎯 守门点：裁判的偏见会污染一切结论

LLM 当裁判（RAG 评估、对比测试、自动打分）已经很普遍，但它有一批已知的"系统性偏见"：

| 偏见 | 表现 |
|---|---|
| **位置偏见** | 倾向于选"排在前面/后面"的那个，而非更好的那个 |
| 长度偏见 | 倾向于选更长、更啰嗦的答案 |
| 自我偏好 | 偏爱与自己同源模型的输出 |
| 不稳定 | 同样输入，多次打分结果飘忽 |

**只要裁判有偏见，它给出的所有评估结论都不可信**——而这恰恰是最隐蔽的：你以为有了量化指标，其实指标本身是歪的。

---

## 🧪 核心方法：考核"位置偏见"（客观可量化）

挑最经典的位置偏见开刀，用客观标准答案来量化：

1. 准备若干**好坏分明**的答案对（一个客观正确、一个客观错误）；
2. 每对**换顺序问裁判两遍**：
   - 顺序 A：好答案放"回答1" → 正确应选 1
   - 顺序 B：好答案放"回答2" → 正确应选 2
3. 看两个指标：
   - **位置一致性**：换顺序后，裁判是否仍选中**同一个真实答案**？（跟着质量走，而非跟着位置走）
   - **准确率**：裁判是否选中了那个**真正更好**的答案？

> 若裁判"两次都选第1个"，位置一致性就会塌——铁证它在看位置而不是看质量。

---

## 📐 风险分级

| 级别 | 触发 | 含义 |
|:--:|---|---|
| 🔴 L1 | 位置一致性 / 准确率 < 80% | 裁判不可信，**其所有评估结论作废**，不得用作门禁 |

> 关键认知：**一个没被考核过的裁判，比没有裁判更危险**——因为它会给你虚假的安全感。
