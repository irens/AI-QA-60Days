# 08 · Agent 工具调用正确性

> Agent 之所以是 Agent，核心就在于会用工具。而工具调用，正是它最容易出错、也最该被守门的环节。

---

## 🎯 守门点：工具调用的三个环节

| 环节 | 出错表现 |
|---|---|
| ① **该不该调** | 闲聊也去乱调工具（误调）；有需求却不调（漏调） |
| ② **选得对不对** | 该查天气却调了计算器（选错工具） |
| ③ **参数准不准** | 从"北京天气"里没抽出 `city=北京`（参数错） |

> 这三个环节任一出错，Agent 的整条行动链就崩了——而且越往后越难定位。

---

## 🧪 核心方法

给模型一份工具清单，让它对每个请求输出 JSON 形式的工具调用决策，再**客观核对**：

- 工具名是否正确（含"闲聊应判 none"）
- 关键参数是否从自然语言里正确抽取

涵盖 3 类请求：**需要工具 A**、**需要工具 B**、**不需要工具（闲聊）**——后者专门测"会不会乱调"。

---

## 📐 门禁

| 级别 | 触发 | 含义 |
|:--:|---|---|
| 🔴 L1 | 工具调用正确率 < 0.8 | Agent 工具决策不可靠，行动链不可信 |
