# Day 62: 优雅降级策略测试

## 🎯 1. 核心风险与测试目标 (20分钟)

> **测试架构师视角**：当系统资源不足或依赖服务不可用时，能否以"有损但可用"的方式继续提供服务？降级策略是否会导致用户体验断崖式下跌？

### 1.1 业务风险点

| 风险等级 | 风险场景 | 业务影响 |
|---------|---------|---------|
| 🔴 L1 阻断性 | 无降级策略，故障时直接拒绝服务 | 用户体验归零，业务损失 |
| 🔴 L1 阻断性 | 降级过度，核心功能不可用 | 用户无法完成关键操作 |
| 🟡 L2 高优先级 | 降级触发条件设置不当 | 过早或过晚触发，资源浪费或体验受损 |
| 🟡 L2 高优先级 | 降级后无法恢复 | 系统长期运行在降级模式 |
| 🟢 L3 一般 | 降级过程用户无感知提示 | 用户困惑，投诉增加 |

### 1.2 测试思路

**核心策略**：验证系统在资源受限或故障场景下的"有损服务"能力

1. **降级触发测试**：验证各种资源阈值下是否正确触发降级
2. **功能保序测试**：验证降级后核心功能仍然可用
3. **用户体验测试**：验证降级后的用户体验下降在可接受范围
4. **自动恢复测试**：验证资源恢复后自动退出降级模式

## 📚 2. 核心理论

### 2.1 优雅降级策略矩阵

```
┌─────────────────────────────────────────────────────────────────┐
│                    优雅降级策略矩阵                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  资源类型    │  正常模式        │  降级模式                      │
│  ───────────┼─────────────────┼────────────────────────────    │
│  LLM模型    │  GPT-4          │  GPT-3.5 → 本地模型 → 规则引擎  │
│  上下文长度 │  128K tokens    │  32K → 8K → 仅保留最近3轮       │
│  检索结果   │  Top-10 + 重排  │  Top-5 → Top-3 → 跳过检索       │
│  响应质量   │  完整推理+引用  │  简化回答 → 仅关键词匹配        │
│  功能范围   │  全功能         │  仅查询 → 仅FAQ → 仅留言        │
│  并发处理   │  全量处理       │  排队+超时 → 采样处理           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 降级触发条件

| 触发类型 | 监控指标 | 阈值示例 | 降级动作 |
|---------|---------|---------|---------|
| **资源阈值** | GPU显存使用率 | >85% | 切换小模型 |
| **响应时间** | P99延迟 | >5s | 简化处理流程 |
| **错误率** | 5xx比例 | >10% | 切换到备用服务 |
| **队列深度** | 等待请求数 | >100 | 限流+快速失败 |
| **依赖故障** | 下游服务健康 | 不可用 | 本地缓存模式 |

### 2.3 LangChain降级策略示例

```python
# 场景1: LLM链降级
Full Chain: Prompt → GPT-4 → Parser → Tool
    ↓ (GPU不足)
Degraded: Prompt → GPT-3.5 → Parser
    ↓ (API限流)
Minimal:  Prompt → 本地模型 → 简单输出
    ↓ (完全不可用)
Emergency: 返回预设回答 + 记录日志

# 场景2: RAG链降级
Full RAG: 向量检索 → 重排序 → 生成回答
    ↓ (向量服务慢)
Degraded: 关键词检索 → 生成回答
    ↓ (检索失败)
Minimal:  仅使用系统Prompt生成通用回答
    ↓ (LLM不可用)
Emergency: 返回"请稍后再试" + 转人工

# 场景3: Agent降级
Full Agent: 规划 → 工具调用 → 观察 → 再规划
    ↓ (工具故障)
Degraded: 简化规划 → 单工具调用
    ↓ (规划失败)
Minimal:  直接回答（无工具）
    ↓ (完全不可用)
Emergency: 返回帮助文档链接
```
