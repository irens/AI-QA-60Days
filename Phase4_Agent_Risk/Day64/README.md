# Day 64: 超时控制与熔断机制

## 🎯 1. 核心风险与测试目标 (20分钟)

> **测试工程师视角**：慢请求拖垮整个系统，级联故障导致雪崩。如果不进行充分测试，线上会出现：
> - 单个慢查询阻塞整个线程池
> - 下游服务故障导致上游服务级联崩溃
> - 系统恢复后流量洪峰再次压垮服务
> - 超时配置不合理导致误杀正常请求

### 1.1 业务风险点

| 风险等级 | 风险场景 | 业务影响 |
|---------|---------|---------|
| 🔴 L1 阻断性 | 级联故障，全系统雪崩 | 业务停摆，收入归零 |
| 🔴 L1 阻断性 | 熔断器失效，持续向故障服务发送请求 | 资源耗尽，无法恢复 |
| 🟡 L2 高优先级 | 超时配置不当，误杀正常请求 | 用户体验下降，业务受损 |
| 🟡 L2 高优先级 | 熔断恢复过快，再次触发故障 | 服务反复抖动 |
| 🟢 L3 一般 | 超时时间设置过长 | 响应慢，体验瑕疵 |

### 1.2 测试思路

**核心策略**：验证超时和熔断机制在各种故障场景下的有效性

1. **超时机制测试**：验证各类超时配置是否生效
2. **熔断器状态转换**：验证OPEN/CLOSED/HALF_OPEN状态转换
3. **级联故障防护**：验证故障隔离是否有效
4. **恢复机制测试**：验证服务恢复后流量逐步放开
5. **半开状态探测**：验证健康检查机制

## 📚 2. 理论基础

### 2.1 熔断器模式 (Circuit Breaker)

```
┌─────────────────────────────────────────────────────────────┐
│                    熔断器状态机                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│    ┌──────────┐    失败率>阈值     ┌──────────┐           │
│    │          │ ─────────────────> │          │           │
│    │  CLOSED  │                    │   OPEN   │           │
│    │ (正常通行)│                    │ (拒绝请求)│           │
│    │          │ <───────────────── │          │           │
│    └────┬─────┘    超时后尝试       └────┬─────┘           │
│         │                                │                 │
│         │ 失败                            │ 成功           │
│         ▼                                ▼                 │
│    ┌──────────┐                      ┌──────────┐         │
│    │ HALF_OPEN│ <─────────────────── │   探测   │         │
│    │ (限流通行)│                      │   请求   │         │
│    └──────────┘                      └──────────┘         │
│                                                             │
│  状态说明：                                                  │
│  - CLOSED: 正常处理请求，统计失败率                          │
│  - OPEN: 快速失败，拒绝请求，触发降级                        │
│  - HALF_OPEN: 放行少量请求探测服务健康状态                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 超时层次设计

| 超时类型 | 作用范围 | 推荐配置 |
|---------|---------|---------|
| 连接超时 | 建立连接 | 1-3秒 |
| 读取超时 | 等待响应 | 5-30秒 |
| 总超时 | 完整请求 | 10-60秒 |
| 业务超时 | 业务逻辑 | 按业务设定 |

### 2.3 测试方法

| 方法 | 适用场景 | 预期结果 |
|-----|---------|---------|
| 延迟注入 | 测试超时触发 | 超时异常抛出 |
| 错误率注入 | 测试熔断触发 | 熔断器打开 |
| 恢复模拟 | 测试熔断恢复 | 状态正常转换 |
| 压力测试 | 测试降级生效 | 系统稳定运行 |
