# Day 56: 上下文管理与长对话稳定性测试

## 🎯 1. 核心风险与测试目标 (20分钟)
> **测试架构师视角**：上下文窗口是有限的资源——长对话中关键信息被淹没、Token超限导致截断、上下文压缩算法引入失真，都会导致Agent"失忆"。我们要验证系统能否在资源约束下保持上下文的完整性和有效性。

### 业务风险点
| 风险场景 | 事故后果 | 发生频率 |
|---------|---------|---------|
| 上下文窗口溢出 | 早期关键信息丢失 | 高频 |
| 上下文压缩失真 | 重要细节被错误摘要 | 中频 |
| 长距离依赖断裂 | 多轮前提到的信息无法引用 | 中频 |
| Token计算错误 | 实际Token超限导致报错 | 低频 |
| 上下文切换混乱 | 话题跳转后上下文未清理 | 中频 |
| 系统消息被挤出 | 关键指令被挤出上下文窗口 | 高风险 |

### 测试思路
1. **窗口边界测试**：验证Token接近上限时的行为
2. **长对话稳定性**：验证50+轮对话后的上下文质量
3. **信息保留策略**：验证关键信息是否被优先保留
4. **话题切换处理**：验证上下文清理和重建
5. **压缩算法验证**：验证摘要压缩后的信息完整性

## 📚 2. 核心理论

### 2.1 上下文窗口管理架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    上下文窗口管理架构                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Token配额分配:                                                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │  系统消息 (System)     │  20%  │  固定保留               │  │
│   ├─────────────────────────────────────────────────────────┤  │
│   │  历史对话 (History)    │  60%  │  动态管理               │  │
│   ├─────────────────────────────────────────────────────────┤  │
│   │  当前输入 (Input)      │  15%  │  必须保留               │  │
│   ├─────────────────────────────────────────────────────────┤  │
│   │  输出预留 (Output)     │   5%  │  为响应预留             │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   历史对话管理策略:                                              │
│   • 滑动窗口: 保留最近N轮                                        │
│   • Token截断: 按Token数截断早期对话                              │
│   • 智能摘要: 对早期对话生成摘要                                  │
│   • 关键信息提取: 保留实体、意图等关键信息                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 上下文管理关键检查点

| 检查点 | 验证内容 | 失败症状 |
|-------|---------|---------|
| **窗口边界** | Token接近上限时是否正确处理 | 报错或截断 |
| **信息保留** | 关键信息是否在长对话中保留 | 早期信息丢失 |
| **系统消息** | 系统指令是否始终保留 | 指令丢失 |
| **话题连贯** | 话题切换后上下文是否正确更新 | 上下文混乱 |
| **压缩质量** | 摘要是否保留关键信息 | 信息失真 |
| **引用准确** | 长距离引用是否正确 | 引用错误 |

### 2.3 长对话中的上下文衰减

```
┌─────────────────────────────────────────────────────────┐
│                  长对话上下文衰减模型                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  对话轮次    信息保留率    风险等级                       │
│  ─────────────────────────────────                      │
│  1-10轮      100%         🟢 安全                       │
│  11-20轮      95%         🟢 安全                       │
│  21-30轮      85%         🟡 注意                       │
│  31-40轮      70%         🟡 注意                       │
│  41-50轮      50%         🔴 警告                       │
│  50+轮        30%         🔴 危险                       │
│                                                         │
│  衰减原因:                                               │
│  1. Token限制导致早期对话被截断                          │
│  2. 摘要压缩导致细节丢失                                 │
│  3. 注意力分散导致关键信息被忽视                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.4 上下文压缩策略对比

| 策略 | 原理 | 优点 | 缺点 | 适用场景 |
|-----|------|-----|------|---------|
| **截断法** | 直接删除早期对话 | 简单快速 | 信息完全丢失 | 短对话 |
| **滑动窗口** | 保留最近N轮 | 实现简单 | 早期信息丢失 | 中等长度对话 |
| **摘要法** | 生成早期对话摘要 | 保留关键信息 | 可能失真 | 长对话 |
| **分层存储** | 关键信息单独存储 | 精确控制 | 实现复杂 | 企业级应用 |
| **向量检索** | 用RAG补充上下文 | 理论上无限 | 延迟增加 | 超长对话 |
