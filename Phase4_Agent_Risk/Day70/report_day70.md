## 🎯 测试场景分析

### 场景定义: 电商订单系统

**实体关系图**:
```
┌─────────┐     ┌─────────┐     ┌───────────┐     ┌─────────┐
│  users  │────<│ orders  │────<│order_items│>────│products │
│  (用户) │  1:N │  (订单) │  1:N │  (订单项) │ N:1 │ (商品)  │
└─────────┘     └─────────┘     └───────────┘     └─────────┘
```

**业务规则**:
1. **订单金额计算**: 订单金额 = Σ(订单项数量 × 商品单价)
2. **用户等级计算**: 根据历史订单总金额自动计算用户等级
3. **订单状态流转**: 待支付 → 已支付 → 已发货 → 已完成
