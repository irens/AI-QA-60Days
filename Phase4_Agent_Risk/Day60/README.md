# Day 60: 组件失败熔断机制测试

## 🎯 1. 核心风险与测试目标 (20分钟)

> **测试架构师视角**：链式流程中单个组件故障可能引发级联崩溃，熔断机制是防止雪崩效应的最后一道防线。

### 1.1 业务风险点

| 风险等级 | 风险场景 | 业务影响 |
|---------|---------|---------|
| 🔴 L1 阻断性 | LLM API超时导致全链路阻塞 | 所有用户请求堆积，系统雪崩 |
| 🔴 L1 阻断性 | 工具调用失败引发无限重试 | 资源耗尽，服务完全不可用 |
| 🟡 L2 高优先级 | 熔断器配置不当（过于敏感或迟钝） | 误伤正常流量或故障扩散 |
| 🟢 L3 一般 | 熔断状态监控缺失 | 故障发现延迟，人工介入滞后 |

### 1.2 测试思路

**核心策略**：以"破坏者"视角验证熔断机制的触发条件和保护效果

1. **故障注入**：模拟组件超时、异常、返回错误
2. **阈值探测**：测试熔断触发阈值（错误率、连续失败次数、响应时间）
3. **状态转换**：验证 CLOSED → OPEN → HALF_OPEN → CLOSED 完整生命周期
4. **边界测试**：测试极端配置下的熔断行为

## 📚 2. 核心理论

### 2.1 熔断器模式（Circuit Breaker Pattern）

```
┌─────────────────────────────────────────────────────────────┐
│                    熔断器状态机                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│    ┌─────────┐    失败率>阈值     ┌─────────┐              │
│    │ CLOSED  │ ─────────────────> │  OPEN   │              │
│    │ (关闭)  │                    │ (开启)  │              │
│    └────┬────┘                    └────┬────┘              │\n│         │                              │                    │
│         │ 请求正常通过                   │ 快速失败，拒绝请求  │
│         │                              │                    │
│         │                              ▼                    │
│         │                         ┌─────────┐              │
│         │      探测成功 <──────── │HALF_OPEN│              │
│         └──────────────────────── │(半开)   │              │
│                                   └─────────┘              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 熔断器核心参数

| 参数 | 说明 | 测试关注点 |
|-----|------|-----------|
| `failure_threshold` | 触发熔断的失败次数阈值 | 阈值边界值测试 |
| `recovery_timeout` | 从OPEN到HALF_OPEN的等待时间 | 时间窗口准确性 |
| `half_open_max_calls` | 半开状态允许的最大探测请求 | 流量控制有效性 |
| `success_threshold` | 半开状态恢复所需的连续成功次数 | 恢复条件合理性 |
| `sliding_window_size` | 统计错误率的滑动窗口大小 | 统计准确性 |

### 2.3 LangChain中的熔断场景

```python
# 典型LangChain链式流程
User Input → Prompt Template → LLM Call → Output Parser → Tool Call → Final Output
                │                │            │             │
                ▼                ▼            ▼             ▼
           [熔断点1]       [熔断点2]     [熔断点3]      [熔断点4]
           模板渲染失败    API超时      解析失败       工具不可用
```

### 2.4 熔断与降级的区别

| 特性 | 熔断 (Circuit Breaker) | 降级 (Fallback) |
|-----|----------------------|----------------|
| **触发条件** | 错误率达到阈值 | 熔断触发或主动选择 |
| **行为** | 快速失败，阻止请求 | 提供替代方案 |
| **目的** | 防止故障扩散 | 保证服务可用性 |
| **关系** | 熔断可以触发降级 | 降级是熔断后的策略 |
