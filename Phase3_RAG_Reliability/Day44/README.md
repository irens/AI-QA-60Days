# Day 44: Ragas框架 - 生成质量指标（Faithfulness + Answer Relevancy）

## 🎯 1. 核心风险与测试目标 (20分钟)
> **测试架构师视角**：生成层的幻觉是RAG系统最大的质量风险，必须量化检测

### 业务风险点
- **事实幻觉**：答案包含上下文未提及的信息，误导用户决策
- **自相矛盾**：答案前后逻辑不一致，损害系统可信度
- **答非所问**：答案与问题无关，用户体验差
- **过度发散**：答案包含过多无关信息，稀释核心价值

### 测试思路
1. **Faithfulness（忠实度）**：验证答案是否忠实于上下文，无幻觉
2. **Answer Relevancy（答案相关性）**：验证答案是否直接回应问题

## 📚 2. 核心理论

### 2.1 Faithfulness（忠实度）

**定义**：答案中的所有事实声明都能在上下文中找到支持的比例

**计算公式**：
```
Faithfulness = 被上下文支持的事实数 / 答案总事实数
```

**与Context Recall的区别**：
| 指标 | 关注点 | 计算方向 |
|------|--------|----------|
| Context Recall | 上下文是否覆盖答案 | 上下文 → 答案 |
| Faithfulness | 答案是否忠实于上下文 | 答案 → 上下文 |

**计算方法（Ragas实现）**：
1. **声明抽取**：将答案分解为独立的事实声明
2. **NLI验证**：使用自然语言推理模型验证每个声明与上下文的关系
   - Entailment（蕴含）：声明被上下文支持 ✓
   - Contradiction（矛盾）：声明与上下文矛盾 ✗
   - Neutral（中立）：无法从上下文推断

**失效模式**：
| 场景 | Faithfulness | 风险 |
|------|--------------|------|
| 答案引入外部知识 | 0.6 | 幻觉风险高 |
| 答案与上下文矛盾 | 0.3 | 严重质量问题 |
| 答案完全基于上下文 | 1.0 | 理想状态 |

### 2.2 Answer Relevancy（答案相关性）

**定义**：答案与问题的语义相关程度，以及答案的完整性

**计算维度**：
1. **主题相关性**：答案是否围绕问题主题展开
2. **意图相关性**：答案是否满足问题的真实意图
3. **完整相关性**：答案是否完整回应问题，无遗漏

**计算方法**：
```
Answer Relevancy = 语义相似度(query, answer) × 完整性系数
```

**与Context Relevancy的区别**：
| 指标 | 关注点 | 评估对象 |
|------|--------|----------|
| Context Relevancy | 上下文与问题的匹配 | 检索层 |
| Answer Relevancy | 答案与问题的匹配 | 生成层 |

**失效模式**：
| 场景 | Relevancy | 风险 |
|------|-----------|------|
| 答案跑题 | 0.3 | 用户体验差 |
| 答案不完整 | 0.6 | 用户需多次追问 |
| 答案过度发散 | 0.5 | 信息噪音大 |

### Ragas Faithfulness实现原理

```python
# Faithfulness 计算伪代码
def faithfulness_score(contexts, answer):
    """
    1. 声明抽取：将答案分解为独立事实
    2. NLI验证：判断每个声明与上下文的关系
    3. 统计支持的声明比例
    """
    claims = extract_claims(answer)  # 例如：["RAG减少幻觉", "RAG支持实时更新"]
    
    supported = 0
    for claim in claims:
        # 使用NLI模型判断
        result = nli_model(claim, contexts)
        if result == "entailment":  # 声明被上下文支持
            supported += 1
        elif result == "contradiction":  # 声明与上下文矛盾
            supported += 0  # 严重问题
        else:  # neutral
            supported += 0.5  # 不确定
    
    return supported / len(claims)
```

### 企业级阈值参考

| 指标 | 优秀 | 合格 | 需优化 |
|------|------|------|--------|
| Faithfulness | > 0.90 | 0.80-0.90 | < 0.80 |
| Answer Relevancy | > 0.85 | 0.70-0.85 | < 0.70 |

### 幻觉检测实战策略

**Level 1 - 规则检测**：
- 检测"我认为""可能"等不确定性词汇
- 检测与上下文直接矛盾的表述

**Level 2 - NLI检测**：
- 使用轻量级NLI模型（如roberta-large-mnli）
- 批量验证声明与上下文的关系

**Level 3 - LLM检测**：
- 使用更强的LLM作为评委
- 针对复杂推理和隐含关系

**Level 4 - 人工抽检**：
- 高风险场景100%人工复核
- 建立幻觉案例库持续优化
