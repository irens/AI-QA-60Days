## 📚 2. 核心理论

### 2.1 语义检索原理
```
用户查询 → Embedding模型 → 查询向量 → 向量相似度计算 → 结果排序
                ↑
         文档库 → Embedding模型 → 文档向量（预计算）
```

### 2.2 向量相似度度量
| 度量方式 | 公式 | 适用场景 |
|---------|------|---------|
| 余弦相似度 | cos(θ) = (A·B) / (\|A\| \|B\|) | 方向比模长更重要 |
| 欧氏距离 | d = \|A - B\| | 绝对距离有意义 |
| 点积 | A·B | 已归一化向量 |

### 2.3 语义检索 vs 关键词检索
| 维度 | 关键词检索 | 语义检索 |
|-----|-----------|---------|
| 匹配方式 | 词汇精确匹配 | 语义相似匹配 |
| 同义词处理 | 需显式扩展 | 自动理解 |
| 概念关联 | 弱 | 强 |
| 计算成本 | 低 | 高 |
| 延迟 | <10ms | 50-200ms |
| 适用场景 | ID、代码、术语 | 开放式问题、概念查询 |

### 2.4 语义检索适用场景
| 场景类型 | 示例 | 为什么适合 |
|---------|------|-----------|
| 同义查询 | "购买" vs "采购" | 语义等价 |
| 概念查询 | "如何提高模型性能" | 多维度相关 |
| 开放式问题 | "介绍一下Transformer" | 语义覆盖 |
| 跨语言 | 中文查询英文文档 | 语义对齐 |

2. **混合检索权重调优**
   ```python
   # RRF (Reciprocal Rank Fusion)
   def rrf_fusion(keyword_results, semantic_results, k=60):
       scores = defaultdict(float)
       for rank, doc_id in enumerate(keyword_results):
           scores[doc_id] += 1.0 / (k + rank + 1)
       for rank, doc_id in enumerate(semantic_results):
           scores[doc_id] += 1.0 / (k + rank + 1)
       return sorted(scores.items(), key=lambda x: x[1], reverse=True)
   ```

### 常见陷阱

| 陷阱 | 表现 | 解决方案 |
|-----|------|---------|
| 向量维度灾难 | 高维向量计算慢 | 降维或量化 |
| 语义漂移 | 相似度高但内容不相关 | 引入重排序模型 |
| 领域错配 | 通用模型在专业领域效果差 | 领域微调或专用模型 |
| 冷启动 | 新文档无向量 | 异步Embedding生成 |
