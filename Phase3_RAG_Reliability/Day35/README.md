# Day 35: 忠实度自动化检测 - 问答事实验证方法

## 🎯 1. 核心风险与测试目标 (20分钟)
> **测试架构师视角**：NLI方法只能检测显式矛盾，问答验证能发现隐式事实错误，两者互补构建完整防线。

### 业务风险点
- **隐式错误**：答案表面通顺，但核心事实错误（如时间、地点、人物混淆）
- **推理错误**：基于上下文的正确推理，但推理过程存在逻辑漏洞
- **信息过时**：答案使用了过时的信息，即使与当前上下文一致也可能错误

### 测试思路
基于**问答（QA）的事实验证方法**：
1. 将每个声明转化为验证问题
2. 让模型基于上下文回答问题
3. 对比答案与原声明的一致性
4. 计算事实一致性分数

## 📚 2. 核心理论

### 2.1 QA-based验证 vs NLI-based验证

| 方法 | 优势 | 局限 | 适用场景 |
|-----|------|------|---------|
| **NLI-based** | 直接、高效、可解释 | 对复杂推理支持有限 | 显式事实验证 |
| **QA-based** | 能处理隐式推理、更灵活 | 需要额外生成步骤 | 复杂事实验证 |

### 2.2 问答验证流程

```
┌─────────────┐    ┌──────────────┐    ┌─────────────┐    ┌─────────────┐
│   Claim     │───→│ Generate     │───→│ Answer with │───→│  Compare    │
│   声明      │    │ Question     │    │ Context     │    │  一致性     │
└─────────────┘    └──────────────┘    └─────────────┘    └─────────────┘
      │                  │                   │                  │
      ▼                  ▼                   ▼                  ▼
 "北京人口     "北京人口多少？"    "约2180万"        一致→支持
  2180万"                                          矛盾→幻觉
```

### 2.3 一致性判定策略

| 判定结果 | 条件 | 处理方式 |
|---------|------|---------|
| **支持** | QA答案与声明一致 | 计入忠实度分子 |
| **矛盾** | QA答案与声明矛盾 | 标记为幻觉 |
| **不确定** | 上下文无法回答 | 标记为Neutral |
| **部分支持** | 部分信息一致 | 按覆盖比例计分 |
