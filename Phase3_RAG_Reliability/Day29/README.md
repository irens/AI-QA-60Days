# Day 29: 领域适配性验证（进阶）

## 🎯 1. 核心风险与测试目标 (20分钟)
> **测试架构师视角**：通用Embedding模型在垂直领域可能"水土不服"，领域适配性决定RAG系统的天花板。

### 业务风险点
- **领域语义鸿沟**：通用模型对医疗、法律等专业术语的理解偏差可达40%，导致检索召回率暴跌
- **术语漂移灾难**：同一术语在不同领域含义完全不同（如"苹果"在科技vs农业），通用模型无法区分
- **长尾实体失效**：专业领域的长尾实体（如罕见疾病名、法律条款编号）在通用模型中向量化质量差
- **领域切换成本**：生产环境切换领域时若无适配验证，可能引发系统性检索失败

### 测试思路
1. **领域特定测试集构建**：针对目标领域构建专业问答对，验证检索准确性
2. **跨领域对比实验**：同一模型在通用领域 vs 垂直领域的表现差异量化
3. **术语对齐验证**：检查领域专业术语的向量空间聚集性
4. **微调效果评估**：领域微调前后的效果提升幅度验证

## 📚 2. 核心理论

### 2.1 领域适配性的核心挑战

| 挑战类型 | 具体表现 | 影响程度 | 检测方法 |
|---------|---------|---------|---------|
| 术语密度差异 | 领域文档术语密度是通用文本的3-5倍 | 🔴 高 | 术语覆盖率测试 |
| 语义专业化 | 通用词在领域内有特殊含义 | 🔴 高 | 一词多义检测 |
| 实体复杂性 | 复合实体、嵌套实体频繁出现 | 🟡 中 | 实体召回测试 |
| 上下文依赖 | 专业概念需要大量前置知识 | 🟡 中 | 上下文完整性测试 |

### 2.2 领域适配策略对比

| 策略 | 实现成本 | 效果提升 | 适用场景 |
|-----|---------|---------|---------|
| 通用模型直接使用 | 低 | 基准 | 通用问答、简单检索 |
| 领域词汇表扩展 | 中 | +5~10% | 术语密集型领域 |
| 对比学习微调 | 高 | +10~20% | 有标注数据的场景 |
| 领域预训练 | 很高 | +15~30% | 大规模领域语料 |
| 混合专家(MoE) | 很高 | +20~35% | 多领域统一服务 |
