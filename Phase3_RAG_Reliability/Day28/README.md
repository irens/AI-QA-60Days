# Day 28: Embedding模型MTEB评测与选型（基础）

## 🎯 1. 核心风险与测试目标 (20分钟)
> **测试架构师视角**：选错Embedding模型，RAG系统从一开始就注定失败——检索质量天花板被锁死。

### 业务风险点
- **检索召回灾难**：通用模型在垂直领域召回率可能低于60%，导致大量用户问题找不到答案
- **语义漂移风险**：不同模型对相同文本的向量表示差异巨大，模型切换可能导致检索结果完全错乱
- **性能成本失控**：高维向量（1024d）存储和计算成本是低维（384d）的3-5倍，但效果提升可能微不足道
- **多语言陷阱**：中文场景使用英文预训练模型，语义理解能力可能下降30%以上

### 测试思路
1. **MTEB基准复现**：验证候选模型在标准评测集上的官方指标是否可信
2. **多维度对比**：从准确性、速度、内存、维度四个维度建立选型矩阵
3. **边界探测**：测试极端长度文本（极短/极长）的编码稳定性
4. **一致性验证**：同一文本多次编码结果的一致性检查

## 📚 2. 核心理论

### 2.1 MTEB（Massive Text Embedding Benchmark）

MTEB是Embedding模型的事实标准评测框架，包含56个数据集、8类任务：

| 任务类型 | 评测重点 | 典型数据集 | 关键指标 |
|---------|---------|-----------|---------|
| 分类（Classification） | 文本类别区分能力 | Banking77, Emotion | Accuracy |
| 聚类（Clustering） | 语义相似文本聚合 | Arxiv, Reddit | V-Measure |
| 语义相似度（STS） | 句子对相似度判断 | STS12-16, BIOSSES | Spearman Correlation |
| 检索（Retrieval） | 从大量文档中找到相关项 | NFCorpus, SciFact | NDCG@10 |
| 重排序（Reranking） | 候选列表精排 | AskUbuntu, SciDocs | MAP |
| 摘要（Summarization） | 摘要与原文匹配 | SummEval | Spearman Correlation |

**关键洞察**：RAG系统最应关注**Retrieval**和**Reranking**任务的表现，而非平均分数。
