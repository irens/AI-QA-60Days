# Day 38: 答案相关性评估 - 意图相关性与完整相关性

## 🎯 1. 核心风险与测试目标（20分钟）
> **测试架构师视角**：即使主题匹配，答案也可能未针对用户的真实意图，或遗漏问题的关键方面。

### 业务风险点
- **意图误解**：用户问"如何退款"，系统回答"退款政策是什么"（主题匹配但意图不符）
- **信息遗漏**：用户问多步骤问题，答案只回答部分步骤
- **答非所问**：用户问比较类问题，答案只描述单方
- **用户挫败**：多次尝试后仍无法获得完整答案，导致放弃使用

### 测试思路
1. **意图识别验证**：检测答案是否针对问题的真实意图（操作vs查询vs比较）
2. **完整性检查**：验证答案是否覆盖问题的所有子问题/方面
3. **问答匹配度**：使用NLI模型判断答案是否直接回应问题
4. **要点覆盖分析**：分解问题要点，检查答案覆盖情况

## 📚 2. 核心理论

### 2.1 意图相关性的定义
意图相关性衡量答案是否针对用户问题的真实意图，而非仅表面主题匹配。

### 2.2 问题意图分类

| 意图类型 | 示例问题 | 期望答案特征 | 常见错误 |
|---------|---------|-------------|---------|
| 操作指导 | "如何安装X？" | 步骤说明、命令示例 | 只解释X是什么 |
| 概念解释 | "什么是Y？" | 定义、原理、示例 | 只列出Y的功能 |
| 比较选择 | "A和B哪个好？" | 对比分析、建议 | 只描述A或B |
| 故障排查 | "为什么报错Z？" | 原因分析、解决方案 | 只解释错误含义 |
| 状态查询 | "当前支持哪些功能？" | 功能列表、版本信息 | 提供过时的信息 |

### 2.3 完整相关性评估

```
问题分解方法：
1. 提取问题中的所有疑问点
2. 识别隐含要求（如"最佳实践"隐含比较）
3. 检查答案对每个疑问点的回应
4. 计算要点覆盖率
```

### 2.4 计算方法

| 方法 | 原理 | 适用场景 |
|-----|------|---------|
| NLI Entailment | 判断答案是否蕴含问题的答案 | 直接问答 |
| 要点匹配 | 分解问题要点，检查覆盖 | 复杂多部分问题 |
| LLM判断 | 询问LLM意图是否匹配 | 复杂意图场景 |
| 用户模拟 | 模拟用户满意度评估 | 端到端评估 |
