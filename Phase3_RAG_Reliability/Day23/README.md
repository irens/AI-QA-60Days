# Day 23: 预处理流程(上) - 文本清洗与格式标准化

## 🎯 1. 核心风险与测试目标
> **测试架构师视角**：未经清洗的原始文档就像未经消毒的手术器械，会将噪声、格式错误和敏感信息带入RAG系统。

### 业务风险点
- **HTML标签残留**：网页抓取的内容包含大量HTML标签，干扰语义理解
- **特殊字符污染**：乱码、控制字符、零宽字符导致解析失败
- **敏感信息泄露**：文档中的手机号、身份证号、API密钥未脱敏
- **格式不统一**：同一知识库中英文标点混用、编码不一致

### 测试思路
- **清洗效果验证**：测试各类清洗规则的有效性
- **信息保留率**：确保清洗过程不丢失关键信息
- **敏感信息检测**：验证脱敏规则的覆盖率
- **格式标准化**：统一编码、标点、换行符

---

## 📚 2. 核心理论

### 2.1 文本清洗流程

```
原始文档
    ↓
[编码检测与转换] → 统一为UTF-8
    ↓
[HTML标签去除] → 提取纯文本
    ↓
[特殊字符清洗] → 移除控制字符、乱码
    ↓
[敏感信息脱敏] → 手机号、身份证、密钥
    ↓
[格式标准化] → 统一标点、换行
    ↓
清洗后文本
```

### 2.2 清洗规则优先级

| 优先级 | 规则类型 | 示例 | 风险等级 |
|-------|---------|------|---------|
| P0 | 敏感信息脱敏 | 手机号→138****8888 | 🔴 高危 |
| P1 | 编码统一 | GBK→UTF-8 | 🟠 中危 |
| P1 | HTML清洗 | `<div>文本</div>`→文本 | 🟠 中危 |
| P2 | 特殊字符 | 移除零宽空格、控制符 | 🟡 低危 |
| P2 | 格式标准化 | 英文标点→中文标点 | 🟡 低危 |

### 2.3 敏感信息脱敏规则

```python
SENSITIVE_PATTERNS = {
    "mobile": (r"1[3-9]\d{9}", "138****8888"),
    "id_card": (r"\d{17}[\dXx]", "110***********0000"),
    "email": (r"[\w.-]+@[\w.-]+\.\w+", "***@***.com"),
    "api_key": (r"sk-[a-zA-Z0-9]{48}", "sk-****************"),
    "bank_card": (r"\d{16,19}", "6222************0000")
}
```
