# Day 27: 递归/Agentic分块策略评估（实战篇）

## 🎯 1. 核心风险与测试目标 (20分钟)
> **测试架构师视角**：递归和Agentic分块是"银弹"还是"过度设计"？成本效益和失败模式是评估关键。

### 业务风险点
| 风险场景 | 线上事故 | 影响程度 |
|---------|---------|---------|
| 递归层级过深 | 处理时间指数级增长，系统超时崩溃 | 🔴 高 |
| Agentic分块成本失控 | LLM调用费用超出预算，API配额耗尽 | 🔴 高 |
| 层级一致性错误 | 父子chunk关系混乱，检索结果不准确 | 🟡 中 |
| Agent决策失败 | LLM输出格式错误，无法解析分块结果 | 🟡 中 |
| 复杂度过高难维护 | 调试困难，问题定位耗时 | 🟡 中 |

### 测试思路
1. **层级深度测试**：测试不同递归深度对性能的影响
2. **成本效益分析**：量化Agentic分块的LLM调用成本与收益
3. **一致性验证**：验证父子chunk关系的正确性
4. **失败模式探测**：模拟LLM输出异常，测试容错能力
5. **综合对比**：三种策略（固定/语义/递归/Agentic）的head-to-head对比

## 📚 2. 核心理论

### 2.1 递归分块原理
```
文档 → 大chunk（如2000字符）
       ↓
   需要细化？ → 中chunk（如500字符）
                 ↓
             需要细化？ → 小chunk（如100字符）
```

**核心思想**：
- 分层处理，从粗粒度到细粒度
- 保留父子关系，支持父文档检索
- 平衡检索精度和上下文完整性

**LangChain推荐**：`RecursiveCharacterTextSplitter` 是新系统默认选择

### 2.2 Agentic分块原理
```
文档 → LLM分析 → 识别结构（章节/段落/列表）
              → 智能分割 → 生成chunks
              → 元数据标注（主题/重要性）
```

**核心思想**：
- 利用LLM理解文档结构
- 智能识别关键信息边界
- 生成带语义标注的chunks

**成本考量**：
- 每文档需要1-N次LLM调用
- 适合高价值文档的精细化处理

### 2.3 四种策略对比

| 策略 | 核心思想 | 优势 | 劣势 | 适用场景 |
|-----|---------|------|------|---------|
| **固定长度** | 按token数分割 | 简单、快速 | 语义断裂 | 结构化文档 |
| **语义分块** | 相似度阈值分割 | 语义连贯 | 阈值敏感 | 叙事性内容 |
| **递归分块** | 分层逐步细化 | 平衡效率质量 | 层级复杂 | 复杂结构文档 |
| **Agentic** | LLM智能分割 | 最优信息组织 | 成本高 | 高价值文档 |

## 📖 扩展阅读

### 分块策略选型决策树

```
文档类型判断
    │
    ├─ 结构化文档（代码/表格）
    │   └─→ 固定长度分块 ✅
    │
    ├─ 叙事性文档（文章/报告）
    │   ├─ 实时性要求高？
    │   │   ├─ 是 → 固定长度分块 ✅
    │   │   └─ 否 → 语义分块 ✅
    │   │
    │   └─ 需要上下文完整性？
    │       ├─ 是 → 递归分块 ✅
    │       └─ 否 → 语义分块 ✅
    │
    └─ 高价值文档（合同/论文）
        ├─ 成本敏感？
        │   ├─ 是 → 递归分块 ✅
        │   └─ 否 → Agentic分块 ✅
        └─ 批处理？
            └─ 是 → 语义分块 + 人工审核 ✅
```

### 混合架构最佳实践

**三层架构**：
```
Layer 1: 固定长度快速分块（所有文档）
    ↓
Layer 2: 语义/递归优化（关键文档）
    ↓
Layer 3: Agentic精细化（高价值文档）
```

**成本分配建议**：
| 文档类型 | 占比 | 推荐策略 | 预算分配 |
|---------|------|---------|---------|
| 普通文档 | 70% | 固定长度 | 10% |
| 重要文档 | 25% | 语义/递归 | 30% |
| 核心文档 | 5% | Agentic | 60% |

### 常见陷阱
1. **过度工程化**：对简单文档使用复杂分块策略
2. **忽视成本**：Agentic分块未做预算控制
3. **层级混乱**：递归分块父子关系维护不当
4. **缺乏回退**：复杂策略失败时无降级方案
5. **忽视监控**：未跟踪分块质量指标

### 关键指标监控

| 指标 | 说明 | 告警阈值 |
|-----|------|---------|
| 分块延迟 | 单文档处理时间 | >5s |
| 块大小方差 | 分块均匀性 | >50% |
| 空块率 | 无效chunk比例 | >5% |
| 父子一致性 | 层级关系错误率 | >1% |
| 成本/文档 | 平均处理成本 | 超预算 |
