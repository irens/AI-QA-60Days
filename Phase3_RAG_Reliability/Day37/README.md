# Day 37: 答案相关性评估 - 主题相关性（Topic Relevance）

## 🎯 1. 核心风险与测试目标（20分钟）
> **测试架构师视角**：答案可能在主题上偏离用户问题，导致"答非所问"，这是RAG系统最常见的用户体验问题之一。

### 业务风险点
- **用户体验受损**：用户询问"Python异步编程"，系统回答"Java多线程"，虽然都是编程主题，但完全偏离用户需求
- **知识库污染**：主题不相关的答案会降低用户对系统的信任度
- **转化率下降**：客服场景中，主题偏离直接导致问题解决失败

### 测试思路
1. **关键词重叠检测**：计算问题与答案的关键词交集
2. **主题分类验证**：使用主题模型判断两者是否属于同一领域
3. **语义相似度**：通过Embedding向量计算主题相关性
4. **边界测试**：测试主题相近但实质不同的场景（如"苹果"vs"苹果手机"）

## 📚 2. 核心理论

### 2.1 主题相关性的定义
主题相关性衡量答案是否涉及问题所关注的主题领域，是答案相关性的基础维度。

### 2.2 计算方法

| 方法 | 原理 | 适用场景 | 局限性 |
|-----|------|---------|--------|
| 关键词重叠 | Jaccard相似度、TF-IDF匹配 | 明确主题词汇的场景 | 无法捕捉语义相似 |
| 主题分类模型 | 预训练分类器判断主题一致性 | 多主题领域系统 | 需要训练数据 |
| Embedding相似度 | 向量空间余弦相似度 | 通用场景 | 对短文本敏感 |
| LLM判断 | 直接询问LLM主题是否相关 | 复杂主题场景 | 成本较高 |

### 2.3 主题偏离的常见模式

```
1. 同词不同义："苹果"（水果）vs "苹果"（公司）
2. 同域不同题："Python异步" vs "Python装饰器"
3. 上位词替代：问具体实现，答概念介绍
4. 相关但偏离：问A产品功能，答B产品功能
```
