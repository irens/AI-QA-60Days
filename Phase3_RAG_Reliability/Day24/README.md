# Day 24: 预处理流程(下) - 完整流水线与质量门禁

## 🎯 1. 核心风险与测试目标
> **测试架构师视角**：预处理流水线是RAG系统的"安检门"，任何一个环节的失效都会导致低质量数据流入知识库。

### 业务风险点
- **流水线单点故障**：某个处理环节失败导致整个文档处理中断
- **异常文档未拦截**：格式损坏、内容为空、超大文档未过滤
- **处理性能瓶颈**：大批量文档处理时耗时过长，影响入库效率
- **质量门禁失效**：低质量文档绕过检测进入知识库

### 测试思路
- **端到端流水线测试**：验证完整预处理流程
- **异常场景覆盖**：空文档、超大文档、损坏文档
- **性能压力测试**：批量处理性能评估
- **质量门禁验证**：阈值设定与拦截效果

## 📚 2. 核心理论

### 2.1 完整预处理流水线

```
┌─────────────────────────────────────────────────────────┐
│                    预处理流水线架构                      │
├─────────────────────────────────────────────────────────┤
│  输入层                                                  │
│  • 文档接收 → 格式识别 → 大小检查                        │
├─────────────────────────────────────────────────────────┤
│  质量门禁层                                              │
│  • 文档质量评分 → 阈值判断 → 拒绝/通过                   │
├─────────────────────────────────────────────────────────┤
│  清洗处理层 (Day 23)                                     │
│  • 编码转换 → HTML清洗 → 特殊字符处理 → 脱敏             │
├─────────────────────────────────────────────────────────┤
│  标准化层                                                │
│  • 格式统一 → 分段处理 → 元数据提取                      │
├─────────────────────────────────────────────────────────┤
│  输出层                                                  │
│  • 质量报告 → 清洗后文档 → 审计日志                      │
└─────────────────────────────────────────────────────────┘
```

### 2.2 质量门禁规则

| 检查项 | 阈值 | 处理方式 |
|-------|------|---------|
| 文档质量分 | < 0.55 | 拒绝入库 |
| 文件大小 | > 100MB | 拒绝处理 |
| 内容为空 | 字符数 < 100 | 拒绝入库 |
| 敏感信息 | 发现未脱敏 | 强制脱敏或拒绝 |
| 重复文档 | 相似度 > 95% | 去重处理 |

### 2.3 异常处理策略

```python
class PreprocessingPipeline:
    def process_document(self, doc):
        try:
            # 1. 输入验证
            if not self.validate_input(doc):
                return {"status": "rejected", "reason": "input_invalid"}
            
            # 2. 质量门禁
            quality_score = self.quality_gate(doc)
            if quality_score < 0.55:
                return {"status": "rejected", "reason": "quality_low"}
            
            # 3. 清洗处理
            cleaned = self.clean_document(doc)
            
            # 4. 标准化
            normalized = self.normalize(cleaned)
            
            return {"status": "success", "doc": normalized}
            
        except Exception as e:
            return {"status": "error", "reason": str(e)}
```
