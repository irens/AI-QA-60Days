# Day 46: LLM-as-a-Judge 评委模型选型与基准校准

## 🎯 1. 核心风险与测试目标 (20分钟)
> **测试架构师视角**：评委模型选错，整个评估体系就崩塌了——Garbage In, Garbage Out。

### 业务风险点
| 风险场景 | 后果 |
|---------|------|
| 评委模型能力不足 | 无法区分高质量与低质量回答，评估失效 |
| 评委与任务不匹配 | 专业领域判断错误，误导优化方向 |
| 缺乏基准校准 | 评分漂移无法发现，长期积累系统性偏差 |
| 单点故障 | 单一评委模型异常导致评估流水线中断 |

### 测试思路
1. **能力基线测试**：验证评委模型在标准评测集上的表现
2. **领域适配性测试**：评估评委在特定业务领域的判断准确性
3. **一致性校准**：多评委投票机制与人机一致性验证
4. **成本效益分析**：评估质量与成本的平衡点

## 📚 2. 核心理论

### 2.1 评委模型选型考量维度

```
┌─────────────────────────────────────────────────────────────┐
│                    评委模型选型决策树                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐  │
│  │  任务复杂度   │───▶│  领域专业性   │───▶│  成本预算     │  │
│  └──────────────┘    └──────────────┘    └──────────────┘  │
│         │                   │                   │          │
│         ▼                   ▼                   ▼          │
│   简单分类→小模型      通用领域→通用模型      高频率→轻量模型 │
│   复杂推理→大模型      专业领域→专用模型      低频率→强模型   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 常见评委模型对比

| 模型 | 能力等级 | 成本 | 适用场景 | 主要局限 |
|-----|---------|------|---------|---------|
| GPT-4 | ⭐⭐⭐⭐⭐ | 高 | 复杂推理、创意评估 | 成本高、延迟大 |
| GPT-3.5 | ⭐⭐⭐ | 中 | 通用分类、简单判断 | 复杂推理不足 |
| Claude 3 | ⭐⭐⭐⭐ | 中高 | 长文本评估、安全判断 | 部分领域偏见 |
| 本地7B模型 | ⭐⭐ | 低 | 高频筛选、初筛过滤 | 准确性有限 |
| 专用微调模型 | ⭐⭐⭐⭐ | 中 | 特定领域评估 | 泛化能力差 |

### 2.3 基准校准方法论

**人机一致性校准流程**：

```python
# 伪代码：人机一致性校准
human_labels = [...]  # 人工标注结果
judge_scores = [...]  # 评委模型评分

# 1. 计算一致性指标
agreement_rate = calculate_agreement(human_labels, judge_scores)
kappa_score = cohen_kappa(human_labels, judge_scores)

# 2. 分析分歧案例
disagreement_cases = find_disagreements(human_labels, judge_scores)
analyze_error_patterns(disagreement_cases)

# 3. 校准评分阈值
calibrated_threshold = optimize_threshold(human_labels, judge_scores)

# 4. 持续监控
monitor_drift(human_labels, judge_scores, window="weekly")
```

### 2.4 多评委投票机制

| 投票策略 | 适用场景 | 优势 | 劣势 |
|---------|---------|------|------|
| 简单多数 | 二分类任务 | 简单高效 | 平票处理 |
| 加权投票 | 评委能力差异大 | 考虑评委权重 | 权重校准复杂 |
| 一致性过滤 | 高质量要求 | 提高可靠性 | 可能丢弃有效样本 |
| 分层评估 | 大规模评估 | 成本可控 | 架构复杂 |
