# Day 39: 答案相关性评估 - 时效相关性（Temporal Relevance）

## 🎯 1. 核心风险与测试目标（20分钟）
> **测试架构师视角**：即使主题、意图都匹配，过时信息也会导致答案失效，特别是在快速变化的技术领域。

### 业务风险点
- **过时指令导致失败**：用户按答案操作后发现命令已废弃
- **错误决策依据**：基于过期数据做出错误业务决策
- **合规风险**：使用过时的法规、政策信息
- **用户信任崩塌**：发现答案包含明显过时的信息

### 测试思路
1. **时间表达式提取**：识别答案中的时间信息（版本号、日期、年份）
2. **时效性验证**：检查时间信息是否在合理范围内
3. **版本兼容性检测**：验证技术答案的版本是否过时
4. **动态内容标记**：识别需要定期更新的内容类型

## 📚 2. 核心理论

### 2.1 时效相关性的定义
时效相关性衡量答案中的时间信息是否符合当前时间要求，包括版本信息、政策有效期、数据时效等。

### 2.2 时效性风险分类

| 风险类型 | 示例 | 影响程度 | 检测方法 |
|---------|------|---------|---------|
| 版本过期 | "Python 2.7的教程" | 🔴 高 | 版本号提取与比对 |
| 政策过期 | "2022年税收政策" | 🔴 高 | 年份提取与当前年比对 |
| 数据陈旧 | "2020年市场份额数据" | 🟡 中 | 数据日期标记 |
| API废弃 | "使用已弃用接口" | 🔴 高 | 弃用标记识别 |
| 预测失效 | "预计2023年发布" | 🟡 中 | 预测时间验证 |

### 2.3 时间表达式类型

```
1. 明确日期："2024年1月"、"January 2024"
2. 版本号："Python 3.8"、"React 16.x"
3. 相对时间："最新版"、"当前"、"现在"
4. 隐含时间："即将发布"、"已废弃"
5. 政策期限："有效期至2025年"
```

### 2.4 时效性评估方法

| 方法 | 原理 | 适用场景 |
|-----|------|---------|
| 规则匹配 | 正则提取时间表达式 | 结构化时间信息 |
| 版本比对 | 与已知最新版本对比 | 技术文档 |
| LLM判断 | 询问时间是否合理 | 复杂时间推理 |
| 外部验证 | 查询官方最新信息 | 关键业务场景 |
