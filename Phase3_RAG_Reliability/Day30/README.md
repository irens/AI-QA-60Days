# Day 30: 向量相似度校准与阈值优化（实战）

## 🎯 1. 核心风险与测试目标 (20分钟)
> **测试架构师视角**：相似度阈值设置不当，直接导致检索系统的精准率-召回率失衡，是RAG系统最常见的隐性故障源。

### 业务风险点
- **阈值过高灾难**：相似度阈值设置过高（如>0.9），导致大量相关文档被过滤，用户问题得不到回答
- **阈值过低污染**：相似度阈值设置过低（如<0.5），引入大量噪声文档，LLM生成质量严重下降
- **模型间不可比**：不同Embedding模型的相似度分布差异巨大，直接复用阈值会导致灾难
- **动态场景失效**：固定阈值无法适应查询难度变化，简单查询和复杂查询使用相同阈值不合理

### 测试思路
1. **相似度分布分析**：分析正例/负例的相似度分布，确定最优阈值区间
2. **P-R曲线绘制**：绘制Precision-Recall曲线，找到最佳平衡点
3. **模型间校准**：建立不同模型间的相似度映射关系
4. **动态阈值策略**：基于查询特征的自适应阈值调整

## 📚 2. 核心理论

### 2.1 相似度阈值的核心矛盾

```
                    相似度阈值权衡
    
    精准率(Precision) ↑                    召回率(Recall) ↑
           │                                  │
    0.9 ───┤  高阈值 → 精准率高              │  高阈值 → 召回率低
           │  但召回率极低                   │  大量相关文档被过滤
           │                                  │
    0.7 ───┤  ⚡ 最佳平衡点                  │  ⚡ 最佳平衡点
           │  精准率↔召回率平衡               │  两者兼顾
           │                                  │
    0.5 ───┤  低阈值 → 精准率低              │  低阈值 → 召回率高
           │  噪声文档混入                   │  但精准率极低
           │                                  │
    0.3 ───┤  极低阈值 → 系统不可用           │  极高阈值 → 系统不可用
           │  大量无关文档                   │  几乎无文档返回
           │                                  │
           └──────────────────────────────────┘
                        阈值设置
```

### 2.2 不同模型的相似度分布特性

| 模型 | 相似度分布特点 | 推荐阈值范围 | 注意事项 |
|-----|--------------|-------------|---------|
| OpenAI text-embedding-3 | 分布较散，高分段稀疏 | 0.70-0.85 | 超过0.9可能无结果 |
| BGE系列 | 分布集中，区分度好 | 0.65-0.80 | 中文场景可适当降低 |
| E5系列 | 余弦相似度偏高 | 0.75-0.90 | 注意内积vs余弦区别 |
| GTE系列 | 分布均匀，可解释性好 | 0.60-0.75 | 适合精细调参 |
| M3E | 中文优化，分布偏右 | 0.70-0.85 | 注意版本差异 |

### 2.3 阈值优化方法论

#### 方法一：基于F1分数最大化
```python
# 寻找使F1最大的阈值
best_threshold = max(
    thresholds,
    key=lambda t: 2 * (precision(t) * recall(t)) / (precision(t) + recall(t))
)
```

#### 方法二：基于业务目标约束
```python
# 约束：召回率不能低于80%
min_recall = 0.80
valid_thresholds = [t for t in thresholds if recall(t) >= min_recall]
best_threshold = max(valid_thresholds, key=lambda t: precision(t))
```

#### 方法三：动态阈值策略
```python
# 根据查询复杂度调整阈值
def adaptive_threshold(query_complexity: float) -> float:
    base_threshold = 0.70
    # 复杂查询降低阈值以保证召回
    # 简单查询提高阈值以保证精准
    return base_threshold - (query_complexity - 0.5) * 0.2
```

### 2.4 模型间相似度校准

当系统需要切换Embedding模型时，必须进行相似度校准：

```
模型A相似度 ──[校准函数]──→ 模型B相似度

校准方法：
1. 分位数校准：将相似度映射到统一分位数
2. 线性校准：y = ax + b
3. Platt缩放：基于逻辑回归的概率校准
```
