# Day 42: 父文档检索器与上下文压缩效率

## 🎯 1. 核心风险与测试目标 (20分钟)
> **测试架构师视角**：小片段检索精度高但缺乏上下文，大段落检索有上下文但噪声多，父文档检索器试图平衡这一矛盾。

### 业务风险点
- **上下文断裂**：小片段检索导致答案缺乏必要的上下文支撑
- **信息碎片化**：用户问题需要跨段落理解，但检索结果是孤立的片段
- **引用不可追溯**：无法追溯到原始完整文档，影响答案可信度
- **压缩过度**：上下文压缩过度导致关键信息丢失

### 测试思路
1. **父文档召回测试**：验证检索小片段后，能否正确召回父级文档
2. **上下文完整性测试**：对比小片段、大段落、父文档三种策略的答案质量
3. **压缩效率测试**：测试不同压缩策略对信息保留率的影响
4. **端到端效果验证**：综合评估父文档检索器在真实场景下的表现

## 📚 2. 核心理论

### 2.1 父文档检索器(Parent Document Retriever)原理

**核心思想**：
1. **检索阶段**：使用小块文本（如句子、段落）进行语义检索，提高匹配精度
2. **召回阶段**：返回小块所属的父级文档（如整页、整节），提供完整上下文
3. **生成阶段**：LLM基于完整父文档生成答案，避免上下文断裂

**优势**：
- 兼顾检索精度（小块匹配）和上下文完整性（大文档生成）
- 可追溯性强，答案引用指向完整文档
- 减少信息碎片化问题

### 2.2 上下文压缩策略

| 策略 | 原理 | 适用场景 | 风险 |
|-----|------|---------|------|
| 句子抽取 | 只保留与查询最相关的句子 | 信息密度高的文档 | 可能丢失上下文逻辑 |
| 摘要生成 | 使用LLM生成文档摘要 | 长文档 | 摘要可能遗漏细节 |
| 关键词过滤 | 只保留含关键词的段落 | 结构化文档 | 可能过滤掉相关但不含关键词的内容 |
| 层次压缩 | 按层次结构逐级压缩 | 有明确结构的文档 | 层次结构识别错误 |

### 2.3 评估指标

```
召回准确率 = 正确召回的父文档数 / 总召回父文档数

上下文完整度 = 答案可追溯到完整文档的比例

压缩信息保留率 = 压缩后保留的关键信息 / 原始关键信息

端到端质量 = 父文档策略答案质量 / 基线策略答案质量
```

### 2.4 三种策略对比

| 策略 | 检索精度 | 上下文完整性 | Token效率 | 适用场景 |
|-----|---------|------------|----------|---------|
| 小片段 | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐⭐ | 事实性问题 |
| 大段落 | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ | 概述性问题 |
| 父文档 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 需要上下文的复杂问题 |
