# Day 36: 忠实度自动化检测 - 声明抽取与综合分析

## 🎯 1. 核心风险与测试目标 (20分钟)
> **测试架构师视角**：声明抽取是忠实度检测的基石，抽取质量直接决定验证准确性。错误的抽取会导致漏检或误报。

### 业务风险点
- **声明遗漏**：复杂答案中的关键声明未被抽取，导致幻觉漏检
- **粒度不当**：声明过粗（包含多个事实）或过细（碎片化）影响验证精度
- **语义扭曲**：抽取过程改变原意，导致后续验证失效

### 测试思路
基于**声明抽取质量评估**的综合分析方法：
1. 评估声明抽取的完整性和准确性
2. 测试不同粒度声明的验证效果
3. 对比多种抽取策略的性能
4. 构建端到端忠实度检测流水线

## 📚 2. 核心理论

### 2.1 声明抽取的核心挑战

| 挑战类型 | 示例 | 影响 |
|---------|------|------|
| **复合声明拆分** | "北京和上海都是一线城市" → 2个声明 | 不拆分会导致部分验证失败 |
| **代词消解** | "它成立于1999年" → 明确主语 | 消解失败导致无法验证 |
| **隐含声明提取** | "因为下雨，所以取消" → 提取因果关系 | 遗漏隐含信息 |
| **否定处理** | "他没有参加会议" → 正确处理否定 | 错误解析导致误判 |

### 2.2 声明粒度与验证效果

```
粒度级别对比：

粗粒度（句子级）：
"北京是中国首都，人口约2180万，GDP约4.4万亿"
→ 验证困难，部分错误难以定位

细粒度（原子级）：
1. 北京是中国首都
2. 北京人口约2180万
3. 北京GDP约4.4万亿
→ 可精确定位错误，但可能丢失关联信息

推荐：中等粒度（事实级）
1. 北京是中国首都，人口约2180万
2. 北京GDP约4.4万亿
→ 平衡完整性和可验证性
```
