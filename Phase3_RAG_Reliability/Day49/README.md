# Day 49: A/B测试流量分配与统计显著性验证

## 🎯 1. 核心风险与测试目标 (20分钟)
> **测试架构师视角**：线上A/B测试若设计不当，会导致错误决策，将劣质模型推向全量用户。

### 业务风险点
| 风险场景 | 潜在损失 | 发生频率 |
|---------|---------|---------|
| 样本量不足即判定胜负 | 误将劣质模型全量上线，用户体验下降 | 高 |
| 流量分配不均（如时间偏倚） | 实验组对照组基线不一致，结果不可信 | 中 |
| 多重比较谬误 | 同时测试多个指标，假阳性率飙升 | 高 |
| 网络效应污染 | 同一用户多次进入不同组，数据混淆 | 中 |
| 早期停止偏倚 | 看到"显著"结果就停止，忽略统计功效 | 高 |

### 测试思路
1. **流量分配验证**：检查哈希分桶是否均匀、是否存在时间/地域偏倚
2. **样本量预估**：验证最小可检测效应(MDE)与统计功效(1-β)的计算
3. **显著性检验**：验证p值计算、置信区间、多重比较校正(Bonferroni/FDR)
4. **早期停止检测**：模拟 peeking 问题，验证序贯检验或固定样本量策略

## 📚 2. 核心理论

### 2.1 A/B测试统计基础

**假设检验框架**：
- H₀: μ_A = μ_B (对照组与实验组无差异)
- H₁: μ_A ≠ μ_B (存在显著差异)
- 显著性水平 α = 0.05 (假阳性率)
- 统计功效 1-β = 0.8 (正确检出真实效应的概率)

**样本量计算公式**（比例指标）：
```
n = 2 × (Z_α/2 + Z_β)² × p(1-p) / δ²
```
其中：
- Z_α/2 = 1.96 (α=0.05双侧)
- Z_β = 0.84 (功效=0.8)
- p = 基准转化率
- δ = 最小可检测效应(MDE)

### 2.2 流量分配策略

| 策略 | 原理 | 适用场景 | 风险 |
|-----|------|---------|------|
| 用户ID哈希 | user_id % 100 | 长期实验 | 新用户无ID |
| Cookie哈希 | cookie % 100 | 匿名用户 | Cookie清除 |
| 会话ID哈希 | session_id % 100 | 短期实验 | 同一用户多会话 |
| 分层分桶 | 多层正交哈希 | 多实验并行 | 层间干扰 |

### 2.3 多重比较问题

当同时监测k个指标时，整体假阳性率：
```
P(至少一个假阳性) = 1 - (1-α)^k
```

**校正方法**：
- **Bonferroni校正**: α' = α/k (保守)
- **Benjamini-Hochberg (FDR)**: 控制假发现率 (推荐)

### 2.4 早期停止偏倚 (Peeking Problem)

**问题**：频繁查看结果并在"显著"时停止，会 inflate Type I error。

**解决方案**：
1. **固定样本量**：实验前确定样本量，达到后才分析
2. **序贯检验**：如 O'Brien-Fleming 边界，允许中期分析
3. **Always Valid P-values**：动态调整显著性边界
