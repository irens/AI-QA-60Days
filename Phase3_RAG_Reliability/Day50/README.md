# Day 50: Embedding漂移检测与线上效果监控体系

## 🎯 1. 核心风险与测试目标 (20分钟)
> **测试架构师视角**：线上数据分布漂移会导致RAG系统性能逐渐衰减，而传统监控无法发现Embedding层面的隐性退化。

### 业务风险点
| 风险场景 | 潜在损失 | 发生频率 |
|---------|---------|---------|
| 概念漂移未检测 | 检索相关性持续下降，用户满意度降低 | 高 |
| 数据分布偏移 | 新类型查询无法被正确编码 | 中 |
| Embedding模型老化 | 领域术语更新导致语义理解偏差 | 高 |
| 监控盲区 | 仅监控延迟/错误率，忽略语义质量 | 高 |
| 告警疲劳 | 阈值设置不当导致大量误报 | 中 |

### 测试思路
1. **漂移检测算法验证**：实现并测试PSI、KL散度、MMD等漂移检测指标
2. **监控指标设计**：定义Embedding质量的关键监控维度
3. **阈值校准**：验证告警阈值的合理性，平衡灵敏度与特异度
4. **根因定位**：设计漂移发生时的诊断流程

## 📚 2. 核心理论

### 2.1 数据漂移类型

| 漂移类型 | 定义 | 对RAG的影响 | 检测难度 |
|---------|------|------------|---------|
| **概念漂移** | P(Y\|X)变化 | 答案相关性下降 | ⭐⭐⭐⭐⭐ |
| **协变量漂移** | P(X)变化 | 检索覆盖率下降 | ⭐⭐⭐⭐ |
| **标签漂移** | P(Y)变化 | 答案分布异常 | ⭐⭐⭐ |
| **特征漂移** | Embedding分布变化 | 语义匹配失效 | ⭐⭐⭐⭐ |

### 2.2 漂移检测指标

**Population Stability Index (PSI)**：
```
PSI = Σ (%Actual_i - %Expected_i) × ln(%Actual_i / %Expected_i)
```
- PSI < 0.1: 无显著漂移
- 0.1 ≤ PSI < 0.25: 轻微漂移
- PSI ≥ 0.25: 显著漂移

**Maximum Mean Discrepancy (MMD)**：
```
MMD² = E[k(X,X')] + E[k(Y,Y')] - 2E[k(X,Y)]
```
- 基于核方法的分布距离度量
- 适用于高维Embedding空间

**KL散度**（Kullback-Leibler Divergence）：
```
KL(P||Q) = Σ P(x) × ln(P(x)/Q(x))
```
- 衡量两个概率分布的差异
- 不对称，需使用JSD变体

### 2.3 监控指标体系

**实时指标**（秒级/分钟级）：
| 指标 | 计算方式 | 告警阈值 |
|-----|---------|---------|
| 查询延迟 | P99响应时间 | > 500ms |
| 错误率 | 失败请求占比 | > 1% |
| QPS | 每秒查询数 | 容量上限80% |

**批量指标**（小时级/天级）：
| 指标 | 计算方式 | 告警阈值 |
|-----|---------|---------|
| Embedding漂移(PSI) | 分布稳定性指数 | > 0.25 |
| 检索相关性 | Top-K命中率 | 下降>5% |
| 答案忠实度 | NLI entailment比例 | 下降>3% |
| 用户满意度 | 点赞/点踩比率 | 下降>10% |
