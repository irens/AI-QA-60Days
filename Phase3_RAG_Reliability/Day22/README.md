# Day 22: 文档质量评分（基础）

## 🎯 1. 核心风险与测试目标 (20分钟)
> **测试架构师视角**：垃圾进，垃圾出（Garbage In, Garbage Out）。文档质量是RAG系统的第一道防线，低质量文档会污染整个知识库。

### 业务风险点
- **格式混乱的文档**：PDF扫描件、图片PDF、乱码文档无法被正确解析，导致检索失败
- **内容不完整的文档**：缺少关键章节、断章取义的文档会提供错误信息，误导用户
- **过时的文档**：政策、价格、流程变更后未及时更新，导致回答过时信息
- **语义不清晰的文档**：口语化严重、逻辑混乱的文档难以被Embedding模型理解，影响检索效果

### 测试思路
- **多维度质量评分**：从格式、内容、时效、语义四个维度评估文档质量
- **质量阈值设定**：建立可量化的质量标准，低于阈值的文档拒绝入库
- **质量分布分析**：分析文档库的整体质量分布，识别质量短板

## 📚 2. 核心理论

### 2.1 文档质量四维评估模型

```
┌─────────────────────────────────────────────────────────┐
│                  文档质量评分体系                        │
├─────────────────────────────────────────────────────────┤
│  格式质量 (Format Quality)        权重: 25%              │
│  • 文件格式支持度  • 解析成功率  • 结构完整性            │
├─────────────────────────────────────────────────────────┤
│  内容质量 (Content Quality)       权重: 35%              │
│  • 信息完整性    • 逻辑连贯性    • 主题聚焦度            │
├─────────────────────────────────────────────────────────┤
│  时效质量 (Timeliness)            权重: 20%              │
│  • 创建时间      • 更新时间      • 过期检测              │
├─────────────────────────────────────────────────────────┤
│  语义质量 (Semantic Quality)      权重: 20%              │
│  • 语言规范性    • 语义清晰度    • 向量化友好度          │
└─────────────────────────────────────────────────────────┘
                    总分 = Σ(维度得分 × 权重)
```

### 2.2 格式质量评分细则

| 检查项 | 评分标准 | 权重 |
|-------|---------|------|
| 文件格式 | PDF/Word/Markdown=1.0, 图片PDF=0.5, 扫描件=0.3 | 30% |
| 解析成功率 | 100%=1.0, 90-99%=0.8, 80-89%=0.6, <80%=0.3 | 40% |
| 结构完整性 | 有标题层级=1.0, 纯文本=0.7, 混乱结构=0.4 | 30% |

**格式质量计算公式**：
```python
format_score = (
    format_type_score * 0.3 +
    parse_success_rate * 0.4 +
    structure_integrity * 0.3
)
```

### 2.3 内容质量评分细则

| 检查项 | 评分标准 | 检测方法 |
|-------|---------|---------|
| 信息完整性 | 关键章节缺失扣分 | 章节标题匹配 |
| 逻辑连贯性 | 段落间连贯度 | 语义连贯性检测 |
| 主题聚焦度 | 离题内容比例 | 主题一致性分析 |

**内容质量计算公式**：
```python
content_score = (
    completeness * 0.4 +      # 完整性
    coherence * 0.35 +        # 连贯性
    focus * 0.25              # 聚焦度
)
```

### 2.4 时效质量评分细则

| 检查项 | 评分标准 | 计算公式 |
|-------|---------|---------|
| 文档年龄 | 越新越好 | `max(0, 1 - days/365)` |
| 更新频率 | 定期更新加分 | 根据领域特点设定 |
| 过期检测 | 检测过期标记 | 关键词匹配 |

**时效质量计算公式**：
```python
timeliness_score = (
    recency * 0.5 +           # 新鲜度
    update_frequency * 0.3 +  # 更新频率
    freshness * 0.2           # 过期状态
)
```

### 2.5 语义质量评分细则

| 检查项 | 评分标准 | 检测方法 |
|-------|---------|---------|
| 语言规范性 | 语法正确性 | 语法检测工具 |
| 语义清晰度 | 信息密度 | 实体密度、关键词密度 |
| 向量化友好度 | 适合Embedding | 句子长度分布、专业术语比例 |

**语义质量计算公式**：
```python
semantic_score = (
    language_quality * 0.3 +  # 语言质量
    clarity * 0.4 +           # 清晰度
    embedding_friendly * 0.3  # 向量化友好度
)
```

### 2.6 综合质量评分与分级

```python
def calculate_overall_score(dimensions: Dict[str, float]) -> float:
    """计算文档综合质量得分"""
    weights = {
        'format': 0.25,
        'content': 0.35,
        'timeliness': 0.20,
        'semantic': 0.20
    }
    
    overall_score = sum(
        dimensions[dim] * weight 
        for dim, weight in weights.items()
    )
    
    return overall_score

# 质量分级标准
def get_quality_level(score: float) -> str:
    if score >= 0.85:
        return "A级-优秀"      # 可直接入库
    elif score >= 0.70:
        return "B级-良好"      # 建议优化后入库
    elif score >= 0.55:
        return "C级-及格"      # 需要人工审核
    else:
        return "D级-不合格"    # 拒绝入库
```
