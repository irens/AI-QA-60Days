# Day 17: Few-shot示例选择与效果稳定性测试

## 🎯 1. 核心风险与测试目标

> **测试架构师视角**：Few-shot示例是Prompt工程的"隐形杠杆"——选得好，模型表现飞跃；选得差，引入偏见与不稳定。我们要验证示例选择的系统性方法，并量化效果稳定性风险。

### 业务风险点

| 风险场景 | 潜在后果 | 测试必要性 |
|---------|---------|-----------|
| 示例与目标任务不匹配 | 模型学习错误模式，输出偏离预期 | 🔴 高 |
| 示例存在隐性偏见 | 模型放大偏见，产生歧视性输出 | 🔴 高 |
| 示例顺序敏感 | 不同排序导致输出不一致 | 🟡 中 |
| 示例数量边界 | 过少无法学习，过多导致上下文溢出 | 🟡 中 |
| 跨模型迁移失败 | 在一个模型上有效的示例，换模型后失效 | 🟡 中 |

### 测试思路

**示例扰动测试**：
- 替换测试：将关键示例替换为低质量/无关示例，观察输出退化程度
- 删除测试：逐步删除示例，测试最少有效示例数
- 重排测试：改变示例顺序，验证输出稳定性

**数量敏感性分析**：
- 0-shot → 10-shot 效果曲线
- 识别"边际收益递减"临界点

**跨模型迁移测试**：
- 同一套示例在不同模型（GPT-3.5/4、Claude、Llama等）上的表现差异

## 📚 2. 核心理论：Few-shot示例选择的系统性方法

### 2.1 示例选择的四维考量

| 考量维度 | 评估方法 | 优化目标 | 风险指标 |
|---------|---------|---------|---------|
| **相似度** | 嵌入空间余弦相似度 | 最大化迁移效果 | 相似度<0.6为高风险 |
| **难度梯度** | 复杂度评分（长度/推理步数） | 促进渐进式学习 | 难度跨度>50%为高风险 |
| **多样性覆盖** | 聚类分析（K-means/Spectral） | 避免偏见和过拟合 | 聚类覆盖率<80%为高风险 |
| **潜在偏见** | 敏感属性分布检查 | 公平性保障 | 敏感属性偏差>20%为高风险 |

### 2.2 效果稳定性测试框架

```
┌─────────────────────────────────────────────────────────┐
│              Few-shot稳定性测试金字塔                    │
├─────────────────────────────────────────────────────────┤
│  Level 3: 跨模型迁移测试                                   │
│  └── 同一示例集在不同LLM上的表现一致性                      │
├─────────────────────────────────────────────────────────┤
│  Level 2: 数量敏感性分析                                   │
│  └── 0-shot → 10-shot 效果曲线与边际收益临界点              │
├─────────────────────────────────────────────────────────┤
│  Level 1: 示例扰动测试                                     │
│  └── 替换/删除/重排对输出质量的影响                         │
└─────────────────────────────────────────────────────────┘
```

### 2.3 关键测试指标

| 指标名称 | 计算方法 | 通过阈值 | 风险等级 |
|---------|---------|---------|---------|
| **扰动稳定性分数** | 1 - (扰动后质量变化 / 原始质量) | >0.8 | L1 |
| **顺序一致性率** | 相同示例不同排序的输出一致性 | >0.9 | L2 |
| **数量效率比** | (k-shot质量 - 0-shot质量) / k | >0.1/k | L2 |
| **跨模型迁移率** | 模型A有效示例在模型B上的有效率 | >0.8 | L3 |
